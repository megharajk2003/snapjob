App Name:
QuickJob (உடனடி வேலை)

Core Idea:
A hyperlocal, real-time job platform for Tamil Nadu, engineered with the speed and map-based UI of Rapido. The primary differentiator is deep cultural integration, robust trust mechanisms, and a user experience optimized for the local market in 2025.

Target Market Context (Chennai, Q4 2025):
User Base: Assumes high smartphone penetration with ubiquitous 5G access. Users are highly proficient with UPI payments (GPay, PhonePe are defaults).

Market Need: A strong demand for flexible gig work exists, but trust is the primary barrier. Users prioritize safety and reliability over the lowest price.

Language: "Tanglish" (Tamil typed in English script) is as common as the Tamil script in daily communication. The app must support all three inputs: English, Tamil, and Tanglish.

Backend Logic & MongoDB Data Architecture
This section specifies the technical implementation for the backend, using a Node.js environment with MongoDB.

1. MongoDB Collections & Schemas
Collection: users
Stores data for both Hirers and Providers in a single collection, differentiated by role.

  "profilePictureUrl": "[https://cdn.quickjob.app/](https://cdn.quickjob.app/)...",
  "languagePreference": "ta", // 'en' or 'ta'
  "isAadhaarVerified": true, // Boolean, Indexed
  "avgRating": 4.8,
  "createdAt": "ISODate",
  "providerProfile": { // Sub-document, exists only if role is 'provider'
    "primarySkill": "Aari Embroidery Blouse Work",
    "secondarySkills": ["Saree Draping", "Mehendi Design"],
    "portfolio": [
      {
        "skill": "Aari Embroidery Blouse Work",
        "mediaUrl": "[https://cdn.quickjob.app/](https://cdn.quickjob.app/)...",
        "mediaType": "video" // 'image' or 'video'
      }
    ],
    "isAvailable": true, // Boolean, critical for map view

Indexes: mobileNumber (unique), currentLocation (2dsphere), isAadhaarVerified.

Indexes: location (2dsphere), status, category.

Collection: applications
Tracks which Providers have applied for which jobs. A lean and efficient linking collection.

{
  "_id": "ObjectId",
  "jobId": "ObjectId(ref:jobs)", // Indexed
  "providerId": "ObjectId(ref:users)", // Indexed
  "hirerId": "ObjectId(ref:users)",
  "status": "pending", // 'pending', 'accepted', 'rejected'
  "appliedAt": "ISODate"
}

Indexes: A compound index on [jobId, providerId].

Collection: reviews
